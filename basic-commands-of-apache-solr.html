Solr's Schema File

All the details about the field types and fields have been stored in the Apache Solr. The file name and file location may vary depending on how we configured our Solr initially or modified it later.

The managed-schema is the schema files Solr uses by default to support making changes in the schema at runtime with the help of Schema API or using the feature of Schemaless Mode. Either we explicitly configure the managed schema features to use an alternative filename, or we choose the contents of the files that are still updated by Solr automatically.

The schema.xml is a schema file that is traditionally named and can be edited manually by users of the ClassicIndexSchemaFactory.

When we use SolrCloud, we may not find any file with these names on the filesystems of the localhost. We can only see the schema with the help of Schema API or using the Solr Admin UI's Cloud Screens.

Whatever the name of the file we use in our installation, the structure of the file remains the same. However, the way we interact with the file will be changed. If we are using the managed schema, it will be expected that we only interact with the file using the Schema API and unable to make manual edits. If we do not use the managed schema, we can only make manual edits to the file, and the Schema API will not support any modifications.
